# Handoff: CHIP! CHIP! (칩칩!) — 브랜드 & UI 디자인 가이드

## Overview
"CHIP! CHIP! 칩칩!"은 컬러칩이 좌우로 스윙하다가 화면을 한 번 터치하면 그대로 낙하해 쌓이는 원터치 스태킹 게임입니다. 이 핸드오프는 기존 프로토타입(https://24onair.github.io/stackup/ 수준의 기본 동작이 구현된 게임)에 적용할 **브랜드·비주얼·모션·사운드 스펙**입니다. 대상 플랫폼: 모바일 세로형 (앱/웹).

## About the Design Files
이 번들의 `.dc.html` 파일들은 **HTML로 제작된 디자인 레퍼런스**입니다 — 의도된 룩앤필과 동작을 보여주는 목업이며, 그대로 복사해 쓸 프로덕션 코드가 아닙니다. 작업은 **이 디자인을 현재 게임 코드베이스의 환경(Canvas/DOM/Phaser 등 기존 구조)에 맞게 재구현**하는 것입니다. 브라우저에서 파일을 직접 열면 전체 가이드를 볼 수 있습니다.

## Fidelity
**High-fidelity (hifi)** — 컬러, 타이포, 간격, 문구가 모두 확정값입니다. 아래 수치를 그대로 적용하세요. 단, 배경 랜드마크 실루엣은 목업에서 CSS 도형으로 그린 참고용이므로, 실제 구현 시 동일한 스타일 규칙(채도 30% 이하, 잉크 외곽선, 단순 기하 형태)을 따르는 아트 에셋으로 제작합니다.

**규칙:** 순수 검정(#000)·순수 흰색(#FFF) 금지. 강조색(#E4572E)은 한 화면에 CTA 1곳만. 그림자는 항상 INK 단색 오프셋(블러 금지, 예: `box-shadow: 4px 4px 0 #33261A`). 외곽선은 2–3px INK.

### 타이포그래피 (Google Fonts)
- **Rammetto One** — 디스플레이(영문·숫자): 로고, 점수, 콤보 문구
- **Jua (주아)** — 한글 헤드라인: 게임 내 모든 한글 문구, 버튼
- **IBM Plex Sans KR** (400/500/700) — 본문, 설정, 설명
- **IBM Plex Mono** (500) — 칩 컬러 코드 라벨

### 타입 스케일
- 점수(SCORE): 64px+ Rammetto One
- 헤드라인: 32–40px Jua
- 버튼: 22–26px Jua
- 본문: 15–16px IBM Plex Sans KR

### 기타
- 하드 섀도: `4px 4px 0 #33261A` (버튼·카드), `3px 3px 0` (작은 배지)
- 그라데이션 금지 (배경 크로스페이드 제외), 라운드는 앱 아이콘·구름 등 한정
- 터치 타깃 최소 44px

## 핵심 오브젝트: 컬러 칩
게임의 시그니처. 항상 두 부분으로 구성:
1. **색면** (위) — 게임 칩 5색 중 하나
2. **크림색 라벨** (아래, 색면 높이의 ~45%) — `CC 165-C 주황` 형식으로 코드+한글 색이름, IBM Plex Mono
- 전체를 3px INK 외곽선으로 감쌈. 낙하 중 칩에는 `4px 4px 0 rgba(51,38,26,0.35)` 그림자.

## Screens / Views

### 1. 게임 플레이 화면
- **HUD (상단):** 좌측 점수 칩(쌓은 칩 개수 + "칩" 단위, 크림 배경/INK 외곽선/하드섀도), 우측 스테이지 배지(청록 배경, "서울 · 스테이지 3" 형식)
- **낙하 칩:** 상단 중앙, 1.6초 주기 좌우 스윙(±26px, ease-in-out)
- **낙하 가이드라인:** 착지점까지 3px 점선, INK 30% 투명도. 설정에서 off 가능(고수 모드)
- **피드백 문구:** 정확히 겹치면 "PERFECT!" 스탬프 — -6° 기울기, Rammetto One, 강조색+INK 그림자, 스택 위 고정, 0.6초 후 소멸
- **스택:** 화면 중앙 하단, 칩끼리 -3px 겹침(외곽선 공유)
- **터치 영역:** 화면 전체. 버튼 없음. 안내 문구("화면을 터치해서 칩을 떨어뜨려요!")는 첫 3회 플레이만
- **배경:** 아래 "스테이지 · 배경 시스템" 참조. 랜드마크는 게임 오브젝트보다 항상 시각적으로 뒤(저채도, 반투명)

### 2. 결과 / 게임오버 화면
- 배경: 강조색 오렌지 풀블리드
- 타이틀: "와르르...!" (부정 언어 금지 — "게임 오버" 대신 의성어)
- **결과 칩 카드:** 최종 점수를 거대한 컬러칩 형태로 — 색면에 점수(72px Rammetto One)+"칩을 쌓았어요", 라벨에 `CC 037-C`(점수 연동 칩 번호) + 스테이지·기록 정보. 공유 이미지로 그대로 사용
- 보조 통계: 퍼펙트 횟수, 최다 콤보 — **2개까지만**
- CTA 위계: "다시 쌓기"(크림색 대형 버튼, 유일한 1순위) / "기록 공유"(청록)·"홈으로"(겨자) 절반 크기
- 진입 모션: 결과 칩이 위에서 "툭" 낙하 → 버튼 순차 슬라이드 업

### 3. 랭킹 화면
- 헤더: 뒤로가기 + "랭킹" 타이틀. 기간 탭 3개(오늘/주간/전체) — 활성 탭만 강조색 오렌지 배경
- **포디움(1–3위):** 컬러칩 형태 — 칩 높이가 순위(1위 최고 높이, 겨자 / 2위 청록 / 3위 벽돌), 라벨에 닉네임 + `CC 112-C · 112칩`(점수 연동 칩 번호). 1위 칩 위에 강조색 삼각 깃발
- **리스트 행(4위~):** 순위(Rammetto) + 최고기록 당시 마지막 칩 색 스와치(20×14px) + 닉네임(Jua) + 우측 "N칩"(Mono)
- **내 순위 행:** 화면 하단 sticky 고정, 유일하게 강조색 3px 외곽선 + 순위 숫자 오렌지
- 진입점: 결과 화면·홈. 신기록 직후 진입 시 내 행 자동 스크롤 + 착지 스쿼시 모션 + VO "NEW RECORD!"

### 4. 스테이지 · 배경 시스템 (핵심 리텐션 장치)
조작이 터치 한 번뿐이므로 **배경이 지루함을 해결**합니다. 나라(도시) 선택 → 랜드마크가 배경 레이어에 배치 → 칩이 쌓일수록 카메라 상승 → 배경이 연속 스크롤.

**서울 스테이지 고도 구간 (Seoul Altitude Scroll.dc.html 참조):**

| 구간 | 칩 범위 | 배경색 | 내용 | 밀도 |
|---|---|---|---|---|
| ZONE 1 지상 | 0–15 | `#EFE3CB` | 광화문 문루, 은행나무, 분식 포장마차, 가로등. 스택 중앙부 폭 140px는 오브젝트 금지 | 100% |
| ZONE 2 스카이라인 | 16–40 | `#EAD9B4` (겨자 노을) | 남산타워(전망대 오렌지) → 63빌딩 → 롯데타워 → 한강 다리, 10칩마다 새 랜드마크 + 이름표 토스트 1초 | 80% |
| ZONE 3 하늘 | 41–70 | `#D9E0D3` | 구름 3종(크림+INK 외곽선 스티커), 5색 줄무늬 열기구, 방패연. 휴식 구간 | 40% |
| ZONE 4 성층권 | 71+ | `#33261A` | 별(2단 도트 패럴랙스), 초승달(겨자), 브랜드 위성 CC-SAT. 전 스테이지 공통 | 25% |

**전환 규칙:**
- 경계 ±3칩 구간에 배경색 세로 그라데이션 프리믹스 → 통과 시 2초 크로스페이드
- 축하 토스트 1.2초: "한강을 건넜어요!" / "서울 하늘 위로!" / "서울 밤하늘을 넘었어요!" + VO "GOING UP!"
- 패럴랙스 3단: 근경 1x / 중경 0.6x / 원경 0.3x

**확장:** 파리(로즈 그레이 `#E8E4D8`, 에펠탑·개선문), 카이로(사막 모래빛 `#F0DEBB`, 피라미드·스핑크스) 등 도시별 시그니처 1색 + 랜드마크 세트로 동일 구조 반복.

## Interactions & Behavior — 모션 스펙
| 모션 | 스펙 | 비고 |
|---|---|---|
| 낙하 | 0.35s, ease-in | 터치 즉시, 지연·연출 없음 (조작감 최우선) |
| 착지 스쿼시 | 0.18s | 가로 110%/세로 88% → 복원, 스택 전체 4px 출렁 |
| 퍼펙트 스탬프 | 0.25s, back-out | 140%→100%, -6° 고정, 연속 퍼펙트 시 5색 순환 |
| 배경 스크롤 | 착지마다 칩 1개 높이 상승, 0.8s ease-out | 3단 패럴랙스 |
| 실패 붕괴 | 0.9s 물리 낙하 | 슬로모션 금지 — 빨리 무너지고 빨리 결과로 |
| 화면 전환 | 0.4s | 5색 칩 띠가 아래→위 와이프, 전 화면 공통 |

## 사운드 스펙 (문구는 한글, 보이스는 영어)
| 시점 | VO | SFX |
|---|---|---|
| 칩 낙하(터치) | — | 짧은 휘파람성 낙하음 0.2s 피치다운 |
| 착지(일반) | — | "톡" 종이카드 폴리 |
| 착지(퍼펙트) | "PERFECT!" | 마림바 단음, 콤보마다 반음 상승 |
| 콤보 x5 | "ON FIRE!" | 브라스 스팅 |
| 고도 구간 전환 | "GOING UP!" | 상승 글리산도 + 앰비언스 교체 |
| 실패 | "OOPS!" | 카드 와르르 폴리 (밝은 톤 유지) |
| 신기록 | "NEW RECORD!" | 팡파레 2초 (유일하게 화려함 허용) |

- BGM: 70년대 라운지/보사노바 루프, BPM 96–108. 스테이지별 악기 1개 추가(서울: 꽹과리 / 파리: 아코디언 / 카이로: 우드 퍼커션)
- 하늘 구간에서 BGM 편성 축소(멜로디+베이스), 바람 앰비언스
- 효과음은 전자음 대신 **종이·나무 질감 폴리** 우선

## State Management (구현 참고)
- `chipCount` (점수 = 쌓은 칩 개수), `perfectCount`, `maxCombo`, `currentCombo`
- `zone` — chipCount에서 파생 (0–15 / 16–40 / 41–70 / 71+), 전환 시 토스트+크로스페이드 트리거
- `stage` — 선택된 도시 (랜드마크 세트·시그니처 색·BGM 악기 결정)
- `bestScore` — 로컬 저장, 갱신 시 NEW RECORD 연출
- 튜토리얼 노출 카운트 (첫 3회)

## Assets
- **`assets/seoul-bg-pixel.png`** — 서울 스테이지 픽셀아트 배경 (720×2880, 원본 144×576 도트의 5배 확대). 세로 스트립 1장에 지상→스카이라인→하늘→성층권 4개 구간이 이어짐. 적용 방법:
  - 반드시 `image-rendering: pixelated`(CSS) 또는 nearest-neighbor 필터로 렌더링, 확대는 정수배만
  - 화면폭에 맞춰 가로 100%로 깔고, `chipCount`에 비례해 세로 스크롤 위치를 위로 이동 (0칩 = 이미지 맨 아래, 최고 고도 = 맨 위)
  - 이 이미지는 통합 레퍼런스 1장 — 여력이 되면 원경(하늘·해·달·별)/중경(빌딩·구름)/근경(지상 오브젝트·다리)으로 레이어를 분리해 0.3x/0.6x/1x 패럴랙스 적용
- 폰트: Google Fonts — Rammetto One, Jua, IBM Plex Sans KR, IBM Plex Mono
- 로고: 워드마크 "CHIP! CHIP!"(Rammetto One, 오렌지+INK 하드섀도) + "칩칩!"(Jua) + 칩 3개 계단 아이콘. 앱 아이콘은 칩 1개 전면 배치
