# CHROMA STACK — 게임 개발 계획서

> 팬톤 컬러칩을 쌓아 올리는 원버튼 물리 밸런스 게임
> 버전 1.0 (프로토타입 동봉) · 2026-07-03

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 한 줄 컨셉 | 움직이는 팬톤 컬러칩을 단 한 번의 탭으로 떨어뜨려, 무너지지 않게 쌓아 올리는 물리 밸런스 게임 |
| 장르 | 하이퍼캐주얼 / 원버튼 스태킹 |

## 2. 핵심 재미 & 차별점

**세 개의 기둥:**

1. **진짜 물리** — 기존 스태킹 게임(Stack 등)은 "삐져나온 부분이 잘리는" 트리밍 방식. CHROMA STACK은 실제 강체 물리(무게중심·토크·마찰)로 칩이 기울고, 미끄러지고, 무너진다. 넘어질 듯 넘어지지 않는 니어미스가 중독의 엔진.
2. **컬러 조형미** — 매 판 다른 팬톤 영감 컬러웨이로 그라데이션 타워가 완성된다. 높이 30 타워의 스크린샷 자체가 포스터가 되도록 설계 → 공유 바이럴 자산.
3. **원버튼** — 조작은 탭 하나. 배우는 데 3초, 마스터하는 데 한 달.

**차별점 요약:** "트리밍의 정확성 게임"이 아니라 "물리의 긴장감 게임". 잘리는 게 아니라 무너진다.

## 3. 코어 루프

```
[관찰] 칩이 한 축으로 등속 왕복 이동
   ↓
[탭] 단 한 번의 클릭 → 칩이 수직 낙하
   ↓
[안착] 물리 시뮬레이션으로 탑 위에 쌓임 (퍼펙트 판정 → 콤보)
   ↓
[긴장] 무게중심 인디케이터가 회색→앰버→레드로 경고
   ↓
[분기] 계속 쌓기 ─ 또는 ─ 붕괴/바닥 낙하 → 게임오버
   ↓
[게임오버] 붕괴 스펙터클(슬로모+줌아웃+탈색) → 결과
   ↓
[재도전] 이어하기(리워드 광고) or 다시하기(1.5초 내 재시작)
```

## 4. 메커니즘 상세

### 4.1 물리 (Matter.js, 시작값)
| 파라미터 | 값 | 의도 |
|---|---|---|
| 칩 크기 | 80×80px (논리 750×1334 기준) | 화면 폭의 ~10.7% |
| friction / frictionStatic | 0.8 / 1.0 | 미끄러짐이 아닌 균형이 스킬이 되도록 |
| restitution | 0.05 | 튕김은 불공정하게 느껴짐 — 거의 데드 바운스 |
| density / frictionAir | 0.002 / 0.02 | 묵직하고 단호한 정착 |
| gravity | 1.2 | 경쾌한 낙하 |
| 타임스텝 | 고정 60fps + 어큐뮬레이터 | 가변 dt는 비결정적 흔들림 유발 |

### 4.2 공정한 전도(넘어짐) 판정
- **무게중심(COM) 인디케이터 상시 표시**: 활성 칩들의 질량가중 무게중심을 지지폭 위에 점선+점으로 투영. 지지폭 대비 60% 이내 회색, 60–85% 앰버, 85%+ 레드 펄스. "왜 무너졌지?"를 "무너질 걸 봤다"로 바꾸는 최우선 공정성 장치.
- **전도 판정**: 35° 초과 기울기가 500ms 지속되거나 두 칸 아래 칩 높이로 추락 → 전도. **2개 이상 동시 전도 또는 최상단 칩 전도 시 게임오버.** 칩 하나가 40°까지 기울었다 복귀하는 건 손실이 아니라 극적인 니어미스.
- **바닥 미스**: 칩이 바닥 센서에 닿는 즉시 게임오버.
- **적층 실패 미스**: 새 칩이 이전 최상단 칩의 윗면 근처(중심 기준 ¼칩 이내)까지 올라가지 못하고 옆으로 안착하면 게임오버 — 타워 옆 플랫폼에 나란히 쌓는 꼼수 차단. 이미 쌓인 칩이 나중에 바로 아래 칩과 같은 높이로 내려앉아도 이탈로 판정.
- **안착 게이트**: 직전 칩이 20프레임 연속 안정(속도<0.15, 각속도<0.02)되거나 2.5초 경과 전까지 다음 칩 미등장 — 출렁이는 타워에 던지게 하지 않는 숨은 공정성.

### 4.3 동결(Freeze) 시스템
위로 8칩이 쌓이고 정지 상태인 칩은 정적 바디로 전환.
- **성능**: 활성 물리 바디 ≤10 유지 → 저사양 폰에서도 높이 200까지 60fps.
- **안정성**: 딥스택의 누적 지터/전단 제거.
- **밸런스(의도된 설계)**: "높이 40에서 초반 실수 하나로 죽는 게임"이 아니라 "항상 최근 8칩이 살아있는" 롤링 난이도 → 더 긴 런 → 더 나은 광고 경제. 인디케이터도 활성 칩 기준으로 계산되므로 물리와 표시가 항상 일치 — 치트처럼 느껴지지 않음.

## 5. 난이도 곡선 — 4단계 이동 페이즈

| 페이즈 | 높이 | 이동 | 스킬 | 기본 속도 |
|---|---|---|---|---|
| 1 · X축 | 1–14 | 좌우 왕복 (±170px) | 수평 조준 | 180px/s |
| 2 · Y축 | 15–29 | 상하 바운스 (낙하고 280–440px) | 착지 충격 관리 — 낮은 지점 릴리즈가 부드러운 착지 | 160px/s |
| 3 · 랜덤 | 30–49 | 칩마다 X/Y 랜덤 (400ms 축 예고 ↔/↕) | 즉각 전환 대응 | +40px/s |
| 4 · 대각선 | 50+ | X+Y 동시 (주기 1.6s/1.13s 리사주 — 패턴 비반복) | 위치+충격 동시 해결 | X260/Y190px/s |

- **속도 램프**: 칩당 +6px/s, 캡 420(X)/320(Y). 삼각파 이동(등속+즉시 반전) — 사인파는 가장자리 타이밍이 뭉개짐.
- **퍼펙트 윈도우**: ±8px(P1) → ±6px(P2–3) → ±4px(P4). Y계열 페이즈는 낮은 지점 릴리즈 조건 추가.
- **점수**: 착지 +10, 퍼펙트 +30×콤보(캡 ×5). **3연속 퍼펙트 → "SET!" 안정화 펄스**(활성 칩 감쇠) — 잘할수록 타워가 진정되는 긍정 피드백 루프.
- **세션 목표**: 첫 세션 중앙값 높이 12–18 (60–90초). 벗어나면 속도 램프 → 진폭 순으로 조정.

## 6. 아트 / 컬러 디렉션

- **팬톤 영감 앵커 팔레트 12색** (Color of the Year 계열: Illuminating, Living Coral, Viva Magenta, Ultra Violet, Classic Blue, Greenery 등)을 색상환 순으로 배열, **OKLCH 색공간 보간**으로 10칩마다 앵커 하나씩 이동 — 높이 60 타워에 6개의 "팬톤 시대"가 담긴다.
- **매 판 랜덤 시작 앵커** → 판마다 다른 컬러웨이 → 스크린샷 다양성.
- **프리미엄 클램프**: L∈[0.62,0.78], C∈[0.08,0.15]. 이 클램프 하나가 "무지개 광대"가 아닌 "팬톤 칩" 룩을 만든다.
- **칩 렌더**: 밝은 상단 에지 + 하단 라벨 밴드(`CHROMA 007`) — 실제 팬톤 칩의 조형 문법.
- **배경 연동**: 현재 칩 hue의 저채도 보색(`oklch 0.955/0.015`)으로 1.5초 크로스페이드 — 오를수록 화면 전체가 조용히 색을 바꾼다. 바닥/텍스트도 hue 연동.
- ⚠️ **상표권**: "Pantone" 명칭·트레이드 드레스(칩 비율+라벨 조합의 직접 모방) 사용 금지. 출시 명칭은 CHROMA 등 자체 브랜드. 라벨 밴드 디자인도 출시 전 법률 검토.

## 7. UX / 주스(Juice)

- **재시작 지연 < 1.5초** — 게임오버 → 탭 → 새 칩 이동까지. 루프 안에 메뉴 없음.
- **퍼펙트**: 화이트 링 확산 + 점수 팝업 + 진동 15ms + 콤보당 펜타토닉 반음 상승음(Web Audio 신스, 에셋 0).
- **니어미스**: 무게중심 레드존 → 심장박동 펄스 + 크리크 사운드(0.6s 스로틀). 20° 이상 기울었다 복귀 → 150ms 마이크로 슬로모 + 휘익.
- **붕괴 스펙터클**: 슬로모 0.25×(400ms) + 셰이크 6px + 바닥 피벗 줌아웃으로 타워 전체 프레이밍 + 1초에 걸쳐 컬러가 "죽는" 탈색 — 조형미가 걸린 판돈임을 각인.
- **신기록**: 경신하는 순간 플레이 중에 축하(팝업+팡파레) — 결과 화면이 아니라.
- **유지**: localStorage — 베스트 점수/높이, 총 판수, 일일 연속 플레이(🔥 스트릭, 타이틀 표시). v1.1에서 스트릭 보상 경제 추가.

## 8. 수익화 설계

| 슬롯 | 위치 | 규칙 |
|---|---|---|
| **리워드 — 이어하기** (주 수익) | 게임오버 결과 화면 | 판당 1회. 시청 → 넘어진 칩 제거 + **생존 타워 전체 동결** 후 재개. 동결은 "이어했는데 바로 또 무너짐" 환불 분노를 원천 차단 |
| **리워드 — 보조** (v1.1) | 결과 화면 / 타이틀 | 점수 2배, 오늘의 컬러웨이 잠금해제 |
| **인터스티셜** | 결과 화면에서 "다시하기" 탭 후 | 3번째 게임오버마다 + 최소 간격 90초 + 세션 첫 게임오버 제외 + 리워드 직후 제외. 붕괴 연출은 절대 끊지 않음 |
| **배너** | — | **사용 안 함.** 제품 정체성인 컬러 조형을 영구 훼손 |
| **IAP** (출시 후) | 설정/상점 | 광고 제거 $2.99 (인터스티셜만 제거, 리워드는 유지) · 컬러웨이 테마팩 $0.99 (앵커 배열 교체만으로 구현 — 개발비 ≈0) |

**구현**: 게임 코드는 `ads.js` 인터페이스(`showRewarded/showInterstitial/canShowInterstitial`)만 호출. 프로토타입은 스텁(가짜 1.5s/1.0s 오버레이)으로 루프 검증 완료, 실제 SDK 교체 시 이 파일 하나만 수정. `?noads` 디버그 파라미터 지원.
